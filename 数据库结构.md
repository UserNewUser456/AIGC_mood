# 数据库结构文档

本文档详细描述了 AIGC 心理陪伴应用的所有数据库表结构，按功能模块分组。

---

## 一、用户模块

### 1.1 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | 主键 | 用户ID |
| username | String(64) | 唯一, 非空 | 用户名 |
| email | String(120) | 唯一 | 邮箱 |
| password_hash | String(128) | 可空 | 密码哈希（匿名用户无密码）|
| user_type | Enum | 默认 anonymous | 用户类型：anonymous / registered |
| role | String(20) | 默认 user | 角色：admin / user |
| is_anonymous | Boolean | 默认 True | 是否匿名用户 |
| avatar_url | String(512) | - | 头像URL |
| created_at | DateTime | - | 创建时间 |
| updated_at | DateTime | - | 更新时间 |

**关系**：
- 一对多：User → EmotionRecord
- 一对多：User → Conversation
- 一对多：User → RiskAlert
- 一对多：User → Order
- 一对多：User → Cart
- 一对多：User → Report

### 2.1 情绪记录表 (emotion_records)

|--------|------|------|------|

## 三、对话模块

### 3.1 对话会话表 (conversations)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | 主键 | 会话ID |
| user_id | Integer | 外键 → users.id, 非空, 索引 | 用户ID |
| session_id | String(64) | 唯一, 索引 | 会话唯一ID |
| doctor_type | String(32) | 默认 gentle | 医生类型：gentle / rational / humorous |
| title | String(200) | - | 会话标题 |
| created_at | DateTime | 索引 | 创建时间 |
| updated_at | DateTime | - | 更新时间 |

**关系**：
- 一对多：Conversation → Message
- 一对多：Conversation → RiskAlert

### 3.2 消息表 (messages)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | 主键 | 消息ID |
| conversation_id | Integer | 外键 → conversations.id, 非空 | 会话ID |
| user_id | Integer | 外键 → users.id, 非空 | 消息所属用户ID |
| role | Enum | 非空 | 角色：user / assistant / system |
| content | Text | 非空 | 消息内容 |
| emotion_detected | String(32) | - | 检测到的情绪 |
| score_detected | Float | - | 检测到的情绪分数 |
| created_at | DateTime | - | 消息时间 |

## 八、表关系总览

```
User (1) ─────< (N) EmotionRecord
User (1) ─────< (N) Conversation (1) ─────< (N) Message >---- (1) User
User (1) ─────< (N) RiskAlert
User (1) ─────< (N) Cart (1) ─────< (N) CartItem >---- (1) Product
User (1) ─────< (N) Order (1) ─────< (N) OrderItem >---- (1) Product
User (1) ─────< (N) Report
```
