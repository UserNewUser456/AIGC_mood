# 基于Neo4j知识图谱的RAG服务部署说明

## 一、系统架构

```
用户提问
    ↓
情绪检测
    ↓
Neo4j知识图谱检索 ← 情绪/症状/治疗方法/技巧
    ↓
AI大模型(qwen-plus)生成回答
    ↓
专业、温暖的心理建议
```

## 二、部署步骤

### 1. 安装Neo4j图数据库

**Linux服务器安装：**

```bash
# 下载Neo4j
wget https://neo4j.com/artifact.php?name=neo4j-community-5.15.0-unix.tar.gz -O neo4j.tar.gz

# 解压
tar -xzf neo4j.tar.gz
mv neo4j-community-5.15.0 /opt/neo4j

# 配置环境变量
echo 'export NEO4J_HOME=/opt/neo4j' >> ~/.bashrc
echo 'export PATH=$PATH:$NEO4J_HOME/bin' >> ~/.bashrc
source ~/.bashrc

# 修改配置（允许远程访问）
sed -i 's/#server.default_listen_address=0.0.0.0/server.default_listen_address=0.0.0.0/' /opt/neo4j/conf/neo4j.conf

# 启动Neo4j
/opt/neo4j/bin/neo4j start

# 查看状态
/opt/neo4j/bin/neo4j status
```

**修改默认密码：**
```bash
# 使用cypher-shell修改密码
/opt/neo4j/bin/cypher-shell -u neo4j -p neo4j

# 在shell中执行：
ALTER USER neo4j SET PASSWORD 'root1234';
:exit
```

### 2. 安装Python依赖

```bash
pip install neo4j flask flask-cors requests
```

### 3. 启动RAG服务

```bash
cd /root/AIGC_mood
python3 my_rag_knowledge_api.py
```

服务将自动：
- 连接Neo4j数据库
- 初始化知识图谱（如果为空）
- 启动API服务（端口5001）

## 三、API接口

### 1. 智能对话（核心接口）
- **URL**: `POST /api/rag-knowledge/chat`
- **请求体**:
```json
{
    "question": "我最近总是很焦虑，晚上睡不着",
    "user_id": 123,
    "stream": false
}
```
- **响应**:
```json
{
    "success": true,
    "data": {
        "question": "我最近总是很焦虑，晚上睡不着",
        "answer": "我理解你现在的焦虑感...",
        "emotion": "焦虑",
        "risk_level": "medium",
        "knowledge_items": [...],
        "knowledge_paths": [...]
    }
}
```

### 2. 获取情绪知识图谱
- **URL**: `GET /api/rag-knowledge/emotion/{情绪名}`
- **示例**: `GET /api/rag-knowledge/emotion/焦虑`
- **响应**: 返回该情绪的症状、原因、治疗方法、技巧

### 3. 搜索知识图谱
- **URL**: `GET /api/rag-knowledge/search?keyword=焦虑`

### 4. 获取所有情绪类型
- **URL**: `GET /api/rag-knowledge/emotions`

### 5. 获取图谱统计
- **URL**: `GET /api/rag-knowledge/graph-stats`

## 四、知识图谱结构

### 节点类型
- **Emotion（情绪）**: 焦虑、抑郁、愤怒、恐惧、孤独、失眠
- **Symptom（症状）**: 心慌、肌肉紧张、注意力难集中、睡眠障碍
- **Treatment（治疗方法）**: 认知行为疗法、正念冥想、运动疗法、深呼吸练习
- **Technique（技巧）**: 4-7-8呼吸法、渐进式肌肉放松、思维记录表
- **Cause（原因）**: 工作压力、人际关系、生活事件等

### 关系类型
- `LEADS_TO`: 情绪导致症状
- `CAUSED_BY`: 情绪由什么原因引起
- `RELIEVED_BY`: 情绪可通过什么方法缓解
- `HAS_TECHNIQUE`: 治疗方法包含什么技巧
- `HAS_SYMPTOM`: 问题有什么症状

## 五、前端调用示例

```javascript
// 智能对话
const res = await fetch('http://49.235.105.137:5001/api/rag-knowledge/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        question: '我最近压力很大，怎么办？'
    })
});
const data = await res.json();
console.log(data.data.answer);

// 获取焦虑的知识图谱
const emotionRes = await fetch('http://49.235.105.137:5001/api/rag-knowledge/emotion/焦虑');
const emotionData = await emotionRes.json();
console.log(emotionData.data.treatments);  // 治疗方法
console.log(emotionData.data.techniques);  // 相关技巧
```

## 六、添加更多知识

可以通过API添加新的知识：

```bash

  -H "Content-Type: application/json" \
  -d '{
    "name": "音乐疗法",
    "description": "通过聆听音乐来调节情绪的方法",
    "for_emotion": "焦虑"
  }'
```

## 七、常见问题

### 1. Neo4j连接失败
- 检查Neo4j是否启动：`neo4j status`
- 检查防火墙是否放行7687端口
- 检查用户名密码是否正确

### 2. 知识图谱为空
- 重启服务会自动初始化基础数据
- 或手动调用初始化接口

### 3. 查询结果为空
- 确认Neo4j中有数据：`MATCH (n) RETURN count(n)`
- 检查情绪名称是否匹配（区分大小写）
